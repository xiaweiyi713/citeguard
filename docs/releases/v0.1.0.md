# CiteGuard v0.1.0

`v0.1.0` is the first alpha release of CiteGuard as an agent-facing skeptical citation auditor.

## Suggested GitHub Repository Description

Short description:

Longer description:

> scholarly records, and whether the evidence supports the claim, with

## Release Summary

CiteGuard v0.1.0 packages the verification core under the public `citeguard.*`
namespace, exposes a `citeguard` CLI and `citeguard-mcp` stdio server, and ships
offline release gates for the public API, MCP contracts, batch workflows, cache
replay, source-outage safety, support-eval provenance, and post-publish smoke
plans.

The product stance is conservative: `not_found`, source outages, sparse metadata,
and possible counter-evidence leads lower confidence or trigger review, but they
are not presented as proof that a citation is fabricated.

## Highlights

- Public `citeguard.*` package structure for verification, retrieval, MCP, CLI,
  and runtime status.
- `citeguard verify`, `audit`, `support`, `support-set`, `support-audit`,
  `counterevidence`, `extract`, `cache`, and `status` command families.
- JSON and JSONL batch audits with `review_summary`, `risk_ranking`, stable
  `next_action`, high-risk filtering, and source-file provenance for extracted
  Markdown, LaTeX/BibTeX/BBL, DOCX, and plain-text references.
- MCP stdio server with status, verification, batch audit, claim-support,
  citation-set support, and counter-evidence tools.
- Offline MCP stdio smoke coverage that initializes a client, lists tools, calls
  status, verifies fixture citations, checks claim-support paths, and validates
  structured expected-error payloads.
- Source-health/status contracts that separate checked, responded, available,
  and failed sources, including retry guidance and outage-safe interpretation.
- Machine-readable error payloads with stable `error.next_action`,
  `error.retryable`, and `error.category` fields for agent branching without
  parsing prose.
- SQLite cache inspect, clear, deterministic export, and offline replay fixture
  support with provenance for source, query, operation, and raw match score.
- Support-eval seed package with hard negatives, contradictions,
  full-text-required boundaries, citation-set policy cases, false-support
  analysis, abstention analysis, and label-sidecar provenance.
- Release checklist and package gates for wheel/sdist contents, public API
  migration, error codes, security/compliance wording, agent-skill examples,
  local wheel MCP stdio smoke, and post-publish PyPI/TestPyPI smoke plans.

## Recommended First Commands

```bash
python -m pip install citeguard
citeguard status
citeguard verify --title "Attention Is All You Need" --author "Ashish Vaswani" --year 2017 --arxiv-id 1706.03762
citeguard audit examples/citations.json
citeguard support-audit examples/claim_citations.json --high-risk-only
python scripts/release_package_gate.py --skip-install-smoke --include-published-smoke-plan --include-published-mcp-smoke-plan
python scripts/release_package_gate.py --skip-install-smoke --include-testpypi-smoke-plan --include-testpypi-mcp-smoke-plan
```

For source checkout release rehearsal, use `python -m pip install -e .` before
running the same CLI commands.

For MCP clients, use Python 3.10+ and install the optional extra:

```bash
python -m pip install "citeguard[mcp]"
citeguard-mcp
python scripts/smoke_mcp.py --require-sdk
```

For source checkout MCP rehearsal, use `python -m pip install -e ".[mcp]"`.

## Release Verification

Before publishing, run the release checklist in `docs/release_checklist.md`.
Important package checks include:

```bash
python scripts/smoke_package.py --install-mode wheel
python scripts/smoke_package.py --install-mode sdist
python scripts/smoke_package.py --install-mode wheel --extra mcp --with-deps --mcp-stdio-smoke
python scripts/release_package_gate.py --require-build-tools --min-high-risk-reviewed-by-language zh=0
```

After publishing to PyPI or TestPyPI, use the post-publish smoke:

```bash
python scripts/smoke_published_package.py --version 0.1.0
python scripts/smoke_published_package.py --version 0.1.0 --extra mcp --require-extra-import mcp --mcp-stdio-smoke --run
```

The post-publish smoke checks installed version consistency, public imports,
console entry points, installed `citeguard --help` / `python -m citeguard --help`
command shape, fixture-backed offline `verify` through both installed CLI launch
paths, fixture-backed offline claim-support through both installed CLI launch
paths, fixture-backed JSONL `audit` / `support-audit --high-risk-only` batch
checks, Markdown extraction plus extracted-reference `audit --high-risk-only`
traceability checks, CLI error-contract checks for `missing_citation_input` and
`invalid_json`, distribution metadata including `License-File: LICENSE`, and
the absence of the legacy `src` namespace in installed artifacts.

For TestPyPI rehearsal before the real PyPI upload, record the dry-run install
plans in the release gate:

```bash
python scripts/release_package_gate.py --skip-install-smoke --include-testpypi-smoke-plan --include-testpypi-mcp-smoke-plan
```

The post-publish smoke runs installed-package checks from an isolated
`smoke-cwd` with `PYTHONPATH` removed, records `config_errors` in JSON, and
fails if configuration errors are non-empty.

## Known Limitations

- This is an alpha auditor package, not a final legal, bibliographic, or research-integrity authority.
- The support-eval seed set is maintainer-authored synthetic data, not a human-reviewed benchmark.
- Support is abstract/metadata-snippet level unless callers provide lawful local
  full-text excerpts or files.
- CiteGuard does not scrape gated sources, bypass paywalls, or crawl CNKI,
  Wanfang, or other protected databases.
- Automated full-text retrieval, full-text multi-hop synthesis across papers,
  contradiction verdicting from counter-evidence leads, and a future large
  human-reviewed benchmark remain future work.
